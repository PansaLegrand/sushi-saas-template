## Pourquoi Google est important pour un SaaS

Google est l’endroit où les gens posent des questions et découvrent des solutions. Pour un SaaS, réussir sur Google permet deux choses :

- Apporter un trafic qualifié et cumulatif depuis la recherche, sur la durée.
- Révéler ce que veulent les utilisateurs et comment ils trouvent votre produit, afin d’améliorer pages, contenus et onboarding.

Pas de “trucs”. Il faut de l’éligibilité, de la découvrabilité, une bonne expérience de page et du contenu utile. Les outils de Google indiquent quoi corriger et comment se comportent les utilisateurs après l’arrivée.

---

## À quoi nous sert Google (la boîte à outils)

### Search Console vs. Google Analytics (GA4)

- Search Console (search.google.com/search-console)
  - Comment vous apparaissez dans Google : requêtes, impressions, clics, position moyenne.
  - Fait remonter les problèmes d’exploration/indexation, Core Web Vitals, sitemaps, actions manuelles, éligibilité aux résultats enrichis.
  - Données issues de la Recherche Google (aucune balise requise). Ne montre pas le trafic hors Recherche ni le comportement sur site.

- Google Analytics 4 (analytics.google.com)
  - Ce que font les utilisateurs sur votre site/app : utilisateurs, sessions, événements, conversions, canaux.
  - Modèle basé sur les événements ; nécessite une balise (Google tag ou via Tag Manager).

Elles sont complémentaires : Search Console explique comment vous obtenez des visiteurs depuis Google ; GA4 explique ce qu’ils font après. On peut visualiser les deux dans Looker Studio.

### Autres surfaces Google utiles

- Google Business Profile (GBP) : visibilité locale dans Maps/requêtes “près de moi” si vous servez une zone.
- Merchant Center + données structurées Produit : éligibilité aux résultats enrichis/Shopping (parcours e‑commerce).
- Trends + Keyword Planner : comprendre la demande, la saisonnalité et les thèmes de requête.
- PageSpeed Insights + Core Web Vitals dans Search Console : signaux de performance réels.

## Playbook pratique pour se classer et croître (2025)

1) Respecter les bases (éligibilité et politiques)
- Suivez Search Essentials et les politiques anti‑spam (pas de liens manipulateurs, ni de contenu peu utile produit à l’échelle pour “ranker”).
- Créez du contenu utile, fiable et orienté utilisateur ; les systèmes de Google récompensent l’utilité, peu importe l’auteur.

2) Assurer la découverte et l’indexation
- Dans Search Console : corrigez les problèmes de l’Indexation des pages, soumettez un sitemap propre et vérifiez que robots.txt/noindex ne bloquent pas les pages clés.
- Utilisez l’Inspection d’URL pour vérifier une page précise et demander un nouveau crawl si nécessaire.

3) Maîtriser l’expérience de page (Core Web Vitals)
- Objectifs : LCP ≤ 2,5s, INP ≤ 200ms, CLS ≤ 0,1 au 75e percentile.
- Utilisez PageSpeed Insights et suivez Core Web Vitals dans Search Console. INP a remplacé FID en mars 2024.

4) Ajouter correctement les données structurées
- Suivez les consignes générales et testez avec Rich Results Test.
- Implémentez les types utiles (Product, Event, Article, Local Business, etc.).
- À noter : HowTo et la plupart des FAQ sont dépréciés/limités ; les variantes de produit sont prises en charge.

5) Renforcer l’architecture d’information
- URLs descriptives, liens explorables avec ancres parlantes, canoniques corrects, hreflang pour sites multilingues.

6) Visibilité locale (si applicable)
- Créez et optimisez votre Google Business Profile (horaires, catégories, photos, services).
- Facteurs locaux : pertinence, distance, notoriété ; de bons avis récents aident.

7) Surfaces e‑commerce (si vous vendez)
- Implémentez les données structurées Product + Merchant listings ; suivez le rapport Merchant listings dans Search Console.
- Créez un compte Merchant Center pour apparaître sur Google (Search/Shopping/Maps/YouTube). Ajoutez éventuellement la politique de retour.

8) Étudier la demande et les sujets
- Utilisez Google Trends pour l’intérêt/saisonnalité ; Keyword Planner pour les thèmes de requêtes (accès complet avec compte Google Ads facturé).

9) Mesurer et itérer
- Dans Search Console : Performance (Search/News/Discover) pour requêtes, CTR, position ; surveillez Crawl Stats et le Search Status Dashboard.
- Dans GA4 : définissez des événements de conversion (leads, achats), comparez acquisition vs. engagement. Si vous opérez dans l’EEE, implémentez le Consent Mode v2.
- Bâtissez un rapport Looker Studio combinant Search Console + GA4.

## Aide‑mémoire (comparatif rapide)

- Quelles requêtes apportent du trafic ? Quel CTR/position ?
  - Search Console → Performance (Search/News/Discover)
- Cette page est‑elle indexée ? Pourquoi/ou non ?
  - Search Console → Inspection d’URL / Indexation des pages
- La page est‑elle rapide/utile pour l’utilisateur ?
  - PageSpeed Insights + Search Console → Core Web Vitals
- Mes produits sont‑ils éligibles aux résultats enrichis/Shopping ?
  - Search Console → Merchant listings + Rich Results Test
- Que font les utilisateurs après l’arrivée ?
  - GA4 (événements/conversions)

## Pièges courants à éviter

- Acheter/échanger des liens manipulateurs ou produire à grande échelle du contenu peu utile (violations anti‑spam). Les mises à jour récentes visent l’“expired domain abuse”, “scaled content abuse” et “site reputation abuse”.
- Miser sur des types de résultats enrichis dépréciés (ex. HowTo ; la plupart des FAQ).
- Ignorer l’INP (remplace FID depuis mars 2024).

---

## Notion clé : qu’est‑ce qu’une « balise » ici ?

En analytique/marketing web, une “balise” est un petit script JavaScript ajouté au site pour envoyer des données (Analytics, Ads, etc.).

- GA4 → nécessite une balise. Exemple : un `<script>` avec l’ID de mesure (G‑XXXXXXX) pour pages, événements, conversions.
- Google Tag Manager (GTM) → conteneur pour ajouter/gérer plusieurs balises (Analytics, Ads, remarketing) sans modifier le code à chaque fois.
- Search Console → ne nécessite pas de balise. On vérifie la propriété une fois (DNS TXT, fichier HTML ou `<meta>` dans `<head>`), puis Google collecte les données de recherche/indexation de son côté.

## Configurer Google Business Profile (SEO local)

Si vous servez une zone géographique ou ciblez Maps :

1. Allez sur https://www.google.com/business/ et connectez‑vous.
2. Renseignez : nom, catégorie, adresse ou zones desservies, contact, horaires, photos.
3. Vérifiez (courrier, téléphone, email ou vidéo).
4. Optimisez : informations à jour, ajout de photos, avis clients, réponses, publications/offres.

## Ordre d’installation pas à pas (site neuf)

1) Search Console
- Vérifiez la propriété (TXT DNS recommandé), soumettez le sitemap et contrôlez l’Indexation des pages.
- Corrigez les blocages (noindex, robots.txt, canoniques) et demandez un recrawl si besoin.

2) GA4 (+ GTM optionnel)
- Ajoutez la balise GA4 (ou passez par GTM). Définissez les conversions (inscription, achat, etc.).
- Si vous opérez dans l’EEE, implémentez le Consent Mode v2.

3) Google Business Profile (si local)
- Créez/réclamez le profil, vérifiez‑le et complétez les champs clés (horaires, services). Commencez à obtenir des avis.

4) Optionnel : Merchant Center (si vous listez des produits)
- Implémentez les données structurées Product/Merchant, connectez Merchant Center et suivez l’éligibilité dans Search Console.

5) Reporting
- Construisez un tableau de bord Looker Studio mélangeant Search Console + GA4 pour suivre requêtes, CTR, conversions et performance des pages.

## Annexe : quand utiliser les outils Google (PME locales)

À utiliser lorsque vous devez :

1) Être découvert — faire indexer les pages, afficher des informations exactes sur Search/Maps, être éligible aux résultats enrichis. (Search Console, Business Profile, données structurées)

2) Mesurer ce qui fonctionne — sources de trafic, événements, conversions, requêtes et expérience de page. (GA4, Search Console, PageSpeed Insights)

3) Améliorer l’expérience — corriger les lenteurs et l’interactivité (Core Web Vitals), tenir à jour horaires/photos/avis. (web.dev, Business Profile)

4) Croître — lancer des annonces, montrer des produits gratuitement sur les surfaces Google, ou utiliser Local Services Ads pour certains métiers. (Google Ads, Merchant Center, LSA)

## Annexe : boîte à outils essentielle (en bref)

- Google Search Console (GSC) — la salle de contrôle SEO
  - Rôle : requêtes/CTR/position, indexation, Core Web Vitals, éligibilité aux rich results ; soumettre des sitemaps ; demander l’indexation ; rapports Merchant/Product.
  - 10 premières minutes : vérifiez la propriété (fichier HTML, DNS TXT ou meta ; GA4/GTM fonctionnent aussi), soumettez le sitemap, inspectez/indexez les URLs prioritaires. Si vous vendez, ouvrez Merchant listings & Product snippets.

- Google Business Profile (GBP) — fiche locale Search & Maps
  - Rôle : publie NAP, horaires, catégories, photos, services/produits, liens de réservation ; recueille des avis ; métriques locales.
  - Notes : chat et historique d’appels retirés le 31/07/2024 ; selon la région, ajoutez un lien WhatsApp/SMS. Éligibilité : contact en personne (pas de boîtes postales/bureaux virtuels). Les entreprises “zone de service” doivent masquer l’adresse et définir des zones.

- Google Analytics 4 (GA4) — mesure orientée événements
  - Rôle : trafic, événements, conversions web/app ; intégration Ads/BigQuery.
  - 10 premières minutes : créez la propriété, installez la balise (ou via GTM), définissez les conversions clés. Si vous servez l’EEE, implémentez Consent Mode v2 (souvent via CMP + GTM).

- Google Tag Manager (GTM) — gérer les balises sans déploiements
  - Rôle : conteneur unique pour déployer/mettre à jour Analytics/Ads/consentement sans sorties de code.

- PageSpeed Insights (PSI) & Core Web Vitals — performance & UX
  - Rôle : données terrain (CrUX) + labo (Lighthouse) pour LCP/INP/CLS. Cibles : LCP ≤ 2,5 s, INP ≤ 200 ms, CLS ≤ 0,1. INP remplace FID depuis 2024.

- Merchant Center — surfaces produits gratuites
  - Rôle : afficher gratuitement les produits sur Search, Shopping, Maps, Images, YouTube, Lens, Gemini ; annonces en option. Alimentez par flux ou données structurées. Côté dev : Product/Offer (livraison/retours, prix/disponibilité) ; suivez GSC Merchant listings.

- Google Ads — croissance payante (option)
  - Rôle : Search/Performance Max/YouTube/Maps ; pour les métiers, Local Services Ads (paiement par lead, avec vérification). Liez GBP pour les extensions de lieu.

- Looker Studio — tableaux de bord
  - Rôle : dashboards gratuits ; connectez GA4 + GSC pour une vue combinée.

- Outils de recherche — Trends & Keyword Planner
  - Rôle : repérer les sujets en hausse et comparer des termes (Trends) ; idées/volumes de mots‑clés (Planner ; accès complet souvent après configuration complète d’Ads).

## Annexe : indispensables vs. optionnels selon le scénario

- Lancement de site (tout type)
  - Indispensables : Search Console (vérification, sitemap, Inspection d’URL), GA4 (conversions de base), correctifs PSI/CWV.
  - Optionnels : dashboards Looker Studio ; mélangez GSC ↔ GA4.

- Commerce local (vitrine ou zone de service)
  - Indispensables : Business Profile (NAP, horaires, catégories, photos, avis) ; facteurs locaux (pertinence, distance, notoriété).
  - Optionnels : liens réservation/commande, menu/produits sur la fiche ; Local Services Ads si éligible.

- Boutique en ligne / catalogue
  - Indispensables : Merchant Center (listings gratuits) + Product/Offer ; suivez GSC Merchant listings.
  - Optionnels : inventaire local (si magasins), Google Ads (Search/PMax).

- Trafic EEE
  - Indispensables : Consent Mode v2 (via CMP/GTM).
  - Optionnels : remarketing avancé respectueux du consentement.

## Annexe : essentiels pour business local (pas à pas)

1) Réclamez et vérifiez votre Business Profile. Ajoutez catégories principale/secondaires, horaires (incl. jours fériés), services, photos, description “À propos”. Pas de boîtes postales/bureaux virtuels.

2) Si vous vous déplacez, définissez des zones de service et masquez l’adresse sur la fiche.

3) Collectez et répondez aux avis. Des avis récents/qualitatifs aident la notoriété ; les réponses montrent l’engagement.

4) Ajoutez des actions utiles : liens de réservation/commande, menu/produits. Note : chat/historique d’appels retirés en 2024 ; ajoutez un lien WhatsApp/SMS si disponible.

5) Option : liez GBP ↔ Google Ads pour afficher l’adresse et l’itinéraire dans les annonces.

6) Suivez “Performance” dans GBP (recherches, vues, appels, itinéraires, clics site).

## Annexe : modules e‑commerce (utiles aussi en local)

- Visibilité gratuite : synchronisez les produits dans Merchant Center pour des listings gratuits sur Search/Shopping/Maps/Images/YouTube/Lens/Gemini.
- Données structurées : implémentez Product/Offer (prix, disponibilité, livraison/retours) ; validez via Rich Results Test ; surveillez GSC Merchant listings.
- Inventaire local / listings locaux gratuits : exposez le stock à proximité et générez des visites en magasin.

---

## Annexe : expérience de page & hygiène technique

- Core Web Vitals : LCP ≤ 2,5 s, INP ≤ 200 ms, CLS ≤ 0,1. Utilisez PSI pour données terrain (CrUX) et labo (Lighthouse).
- Sitemap : générez/soumettez un XML standard ; soignez le maillage interne.
- Robots & indexation : robots.txt gère l’exploration, pas l’indexation ; utilisez noindex ou l’auth pour exclure des pages.
- Dépannez les pages avec Inspection d’URL (test en direct, demande d’indexation).
- En cas de volatilité, vérifiez le Search Status Dashboard (mises à jour majeures/incidents).

---

## Annexe : recherche & planification (workflow léger)

- Commencez par Trends pour vocabulaire, saisonnalité et géos ; comparez jusqu’à 5 groupes de termes.
- Utilisez Keyword Planner pour estimer la demande et trouver des thèmes liés (certaines fonctions requièrent la configuration complète d’Ads).
- Pour la qualité de contenu, suivez “helpful, reliable, people‑first content” et Search Essentials.

## Checklist d’une page (nouveau site)

Jours 1–2
- Vérifiez Search Console (fichier HTML, DNS TXT ou meta). Soumettez sitemap.xml. Inspectez quelques URLs clés.
- Installez GA4 (gtag ou via GTM). Configurez 1–3 conversions (lead, achat, clic d’appel).
- Lancez PageSpeed Insights sur la page d’accueil et les gabarits clés ; notez les points LCP/INP/CLS.

Semaine 1
- Business Profile : réclamer/vérifier ; catégories, horaires (incl. fériés), zones de service (si applicable), description, photos ; ajoutez des liens de réservation/commande si utile. Lancez un plan d’avis.
- Si vous vendez en ligne : ajoutez Product/Offer ; validez via Rich Results Test ; ouvrez GSC Merchant listings.

Mois 1
- Bâtissez un dashboard Looker Studio combinant GA4 (engagement/conversions) + GSC (requêtes/positions).
- Si vous servez l’EEE, déployez Consent Mode v2 avec votre CMP via GTM.
- Envisagez Local Services Ads (métiers) ou un test Search Ads ; liez Ads ↔ GBP.

## Glossaire

- Balise (tag) — petit code chargé par le site (Analytics, Ads, etc.). GTM aide à gérer sans déploiements.
- Données structurées / schéma — info lisible par machine (JSON‑LD recommandé) pour activer des rich results (ex. prix/notes). Testez avec Rich Results Test.
- Core Web Vitals (CWV) — LCP, INP, CLS. Visez les seuils “bons”.
- Sitemap — liste d’URLs importantes ; à soumettre dans GSC.
- Robots.txt — fichier qui gère l’exploration ; n’empêche pas l’indexation (utilisez noindex ou l’auth).

## Pensée finale

Google reste la porte d’entrée principale du web, surtout pour la découverte locale. Si vous gagnez en visibilité (contenu utile, site sain), mesurez correctement (GA4/GSC), améliorez l’expérience (CWV) et enrichissez les résultats (données structurées, Business Profile, Merchant Center), vous couvrez l’essentiel (80/20) d’une croissance durable ; la pub et le reporting avancé peuvent venir ensuite.
