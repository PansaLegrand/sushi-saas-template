## Por qué Google importa para un SaaS

Google es donde la gente hace preguntas y descubre soluciones. Para un SaaS, ganar en Google logra dos cosas:

- Atrae tráfico cualificado y acumulativo desde la búsqueda con el tiempo.
- Revela qué quieren los usuarios y cómo encuentran tu producto, para mejorar páginas, contenido y onboarding.

No necesitas trucos. Necesitas elegibilidad, descubribilidad, buena experiencia de página y contenido útil. Las propias herramientas de Google te dicen qué corregir y cómo se comportan los usuarios una vez que llegan.

---

## Para qué usamos Google (el toolkit)

### Search Console vs. Google Analytics (GA4)

- Search Console (search.google.com/search-console)
  - Cómo apareces en Google: consultas, impresiones, clics, posición media.
  - Señala problemas de rastreo/indexación, Core Web Vitals, sitemaps, acciones manuales, elegibilidad para datos enriquecidos.
  - Los datos vienen de la Búsqueda de Google (no requiere tag). No muestra tráfico no proveniente de Búsqueda ni el comportamiento en el sitio.

- Google Analytics 4 (analytics.google.com)
  - Qué hacen los usuarios en tu sitio/app: usuarios, sesiones, eventos, conversiones, canales.
  - Modelo basado en eventos; requiere una etiqueta (Google tag o mediante Tag Manager).

Son complementarias: Search Console explica cómo llegan los visitantes desde Google; GA4 explica qué hacen después. Puedes visualizar ambas en Looker Studio.

### Otras superficies de Google que nos importan

- Google Business Profile (GBP): visibilidad local en Maps/búsquedas “cerca de mí” si sirves una zona.
- Merchant Center + datos estructurados de producto: elegibilidad para resultados enriquecidos/Shopping (flujos e‑commerce).
- Trends + Keyword Planner: entender demanda, estacionalidad y temas de consulta.
- PageSpeed Insights + Core Web Vitals en Search Console: señales de rendimiento de usuarios reales.

## Playbook práctico para posicionar y crecer (2025)

1) Cumple lo básico (elegibilidad y políticas)
- Sigue Search Essentials y las políticas anti‑spam (nada de enlaces manipulados ni contenido masivo poco útil/solo para rankear).
- Crea contenido útil, fiable y centrado en las personas; los sistemas de Google recompensan la utilidad independientemente de quién/cómo se escriba.

2) Asegura que Google puede encontrar e indexar
- En Search Console: corrige problemas del informe de Indexación de páginas, envía un sitemap limpio y confirma que robots.txt o noindex no bloquean páginas clave.
- Usa Inspección de URL para revisar páginas concretas y (cuando proceda) solicitar un nuevo rastreo.

3) Clava la experiencia de página (Core Web Vitals)
- Objetivos: LCP ≤ 2.5s, INP ≤ 200ms, CLS ≤ 0.1 en el percentil 75.
- Usa PageSpeed Insights y monitoriza Core Web Vitals en Search Console. Nota: INP sustituyó a FID en marzo de 2024.

4) Añade datos estructurados correctamente
- Sigue las directrices generales y prueba con Rich Results Test.
- Implementa los tipos que importan (Product, Event, Article, Local Business, etc.).
- Cambios recientes: HowTo y la mayoría de FAQ están depreciados/limitados; variantes de producto tienen soporte.

5) Refuerza la arquitectura de información
- URLs descriptivas; enlaces rastreables con anclajes significativos; canónicos correctos; hreflang para sitios multilenguaje.

6) Visibilidad local (si aplica)
- Crea y optimiza tu Google Business Profile (horarios, categorías, fotos, servicios).
- Factores de ranking local: relevancia, distancia, prominencia; reseñas recientes y positivas ayudan.

7) Superficies e‑commerce (si vendes online)
- Implementa datos estructurados de Product y Merchant listings; monitoriza el informe de Merchant listings en Search Console.
- Abre una cuenta de Merchant Center para aparecer en Google (Search/Shopping/Maps/YouTube). Añade la política de devoluciones opcionalmente.

8) Investiga demanda y temas
- Usa Google Trends para interés/estacionalidad; usa Keyword Planner para validar temas de consulta (acceso completo con cuenta de Google Ads con facturación).

9) Mide e itera
- En Search Console: usa los informes de Rendimiento (Search/News/Discover) para consultas, CTR y posición; revisa Crawl Stats y el Search Status Dashboard.
- En GA4: define eventos de conversión (leads, compras), analiza adquisición vs. engagement. Si operas en el EEE, implementa Consent Mode v2.
- Construye un informe sencillo en Looker Studio mezclando Search Console + GA4.

## Comparativa rápida (chuleta)

- ¿Qué consultas traen tráfico? ¿CTR/posición?
  - Search Console → Rendimiento (Search/News/Discover)
- ¿Esta página está indexada? ¿Por qué sí/no?
  - Search Console → Inspección de URL / Indexación de páginas
- ¿Qué tan rápida/usable es la página?
  - PageSpeed Insights + Search Console → Core Web Vitals
- ¿Mis productos son elegibles para resultados enriquecidos/Shopping?
  - Search Console → Merchant listings + Rich Results Test
- ¿Qué hacen tras aterrizar?
  - GA4 (eventos/conversiones)

## Errores comunes a evitar

- Comprar/participar en enlaces manipulados o producir contenido poco útil a escala (viola políticas de spam). Actualizaciones recientes atacan el “abuso de dominios expirados”, “abuso de contenido a escala” y “abuso de reputación del sitio”.
- Confiar en tipos de resultados enriquecidos depreciados (p. ej., HowTo; la mayoría de FAQ).
- Ignorar INP (sustituyó a FID en marzo de 2024).

## Concepto clave: ¿Qué es una “tag” aquí?

En analítica/marketing web, una “tag” es un pequeño fragmento de JavaScript que añades al sitio para enviar datos a un servicio (Analytics, Ads, etc.).

- GA4 → requiere etiqueta. Ejemplo: `<script>` con un Measurement ID (G‑XXXXXXX) para páginas, eventos y conversiones.
- Google Tag Manager (GTM) → contenedor para añadir/gestionar muchas etiquetas (Analytics, Ads, remarketing) sin tocar el código cada vez.
- Search Console → no requiere etiqueta. Verificas la propiedad una vez (DNS TXT, archivo HTML o `<meta>` en `<head>`) y Google recolecta los datos de búsqueda/indexación por su lado.

## Cómo configurar Google Business Profile (SEO local)

Si sirves una ubicación o quieres visibilidad en mapas:

1. Ve a https://www.google.com/business/ e inicia sesión.
2. Añade los datos del negocio: nombre, categoría, dirección o áreas de servicio, contacto, horarios, fotos.
3. Verifica (postal, teléfono, email o video).
4. Optimiza: mantén la información actualizada, añade fotos, solicita reseñas, respóndelas y publica novedades/ofertas.

## Orden paso a paso (sitio nuevo)

1) Search Console
- Verifica la propiedad (preferible DNS TXT), envía el sitemap y revisa Indexación de páginas.
- Corrige bloqueos (noindex, robots.txt, canonicals) y solicita recrawls cuando aplique.

2) GA4 (+ opcional GTM)
- Añade la etiqueta de GA4 (o instala vía GTM). Define eventos de conversión (registro, compra, etc.).
- Si operas en el EEE, implementa Consent Mode v2.

3) Google Business Profile (si es local)
- Crea/reclama el perfil, verifica y completa campos clave (horarios, servicios). Empieza a construir reseñas.

4) Opcional: Merchant Center (si listás productos)
- Implementa datos estructurados de Product/Merchant, conecta Merchant Center y monitoriza la elegibilidad en Search Console.

5) Reporting
- Monta un dashboard básico en Looker Studio combinando Search Console + GA4 para monitorear consultas, CTR, conversiones y rendimiento por página.

## Anexo: ¿Cuándo usar las herramientas de Google? (pymes locales)

Úsalas cuando necesites:

1) Ser descubierto — lograr indexación, mostrar información exacta en Search/Maps, ser elegible para resultados enriquecidos. (Search Console, Business Profile, datos estructurados)

2) Medir qué funciona — fuentes de tráfico, eventos, conversiones, consultas y experiencia de página. (GA4, Search Console, PageSpeed Insights)

3) Mejorar la experiencia — corregir lentitud/INP alto (Core Web Vitals), mantener horarios/fotos/reseñas al día. (web.dev, Business Profile)

4) Crecer — ejecutar anuncios, mostrar productos gratis en superficies de Google, o activar Local Services Ads en oficios elegibles. (Google Ads, Merchant Center, LSA)

## Anexo: Caja de herramientas esencial (de un vistazo)

- Google Search Console (GSC) — tu sala de control SEO
  - Qué hace: consultas/CTR/posición, indexación, Core Web Vitals, elegibilidad de rich results; enviar sitemaps; solicitar indexación; informes de Merchant/Product.
  - Primeros 10 minutos: verifica propiedad (archivo HTML, DNS TXT o meta; GA4/GTM también valen), envía el sitemap, inspecciona/indexa URLs prioritarias. Si vendes productos, abre “Merchant listings” y “Product snippets”.

- Google Business Profile (GBP) — ficha local en Search y Maps
  - Qué hace: publica NAP, horarios, categorías, fotos, servicios/productos, enlaces de reservas; recoge reseñas; muestra métricas locales.
  - Notas: chat e historial de llamadas se retiraron el 31/07/2024; en algunas regiones puedes añadir enlaces de WhatsApp/SMS como reemplazo. Debes tener contacto en persona (sin apartados postales/oficinas virtuales). Los negocios de área de servicio deben ocultar la dirección y fijar áreas.

- Google Analytics 4 (GA4) — medición basada en eventos
  - Qué hace: tráfico, eventos y conversiones web/app; integra con Ads/BigQuery.
  - Primeros 10 minutos: crea la propiedad, instala la etiqueta (o vía GTM), define conversiones clave. Si atiendes al EEE, implementa Consent Mode v2 (a menudo vía CMP + GTM).

- Google Tag Manager (GTM) — gestiona etiquetas sin despliegues de código
  - Qué hace: un contenedor para desplegar/actualizar etiquetas de Analytics/Ads/consentimiento sin tocar código.

- PageSpeed Insights (PSI) y Core Web Vitals — salud de rendimiento/UX
  - Qué hace: datos de campo (CrUX) + laboratorio (Lighthouse) para mejorar LCP/INP/CLS. Objetivos: LCP ≤ 2.5s, INP ≤ 200ms, CLS ≤ 0.1. INP sustituyó a FID en 2024.

- Merchant Center — superficies de producto gratuitas
  - Qué hace: muestra productos gratis en Search, Shopping, Maps, Images, YouTube, Lens y Gemini; anuncios opcionales. Funciona con feeds o datos estructurados. Para devs/SEOs: añade Product/Offer (envío/devoluciones, precio/disponibilidad); monitoriza Merchant listings en GSC.

- Google Ads — crecimiento de pago (opcional)
  - Qué hace: Search/Performance Max/YouTube/Maps; para oficios, Local Services Ads (pago por lead, con verificación). Vincula GBP para extensiones de ubicación.

- Looker Studio — dashboards de reporting
  - Qué hace: paneles gratis; conecta GA4 + GSC para visualizar juntos.

- Investigación — Trends y Keyword Planner
  - Qué hace: detectar temas en alza y comparar términos (Trends); ideas/volúmenes de keywords (Planner; acceso completo suele requerir cuenta de Ads configurada).

## Anexo: Imprescindibles vs. recomendables por escenario

- Cualquier lanzamiento de web
  - Imprescindible: Search Console (verificar, sitemap, Inspección de URL), GA4 (conversiones básicas), correcciones de PSI/Core Web Vitals.
  - Recomendable: dashboards en Looker Studio; combinar GSC ↔ GA4.

- Negocio local (tienda o área de servicio)
  - Imprescindible: Business Profile (NAP, horarios, categorías, fotos, reseñas); factores locales (relevancia, distancia, prominencia).
  - Recomendable: enlaces de reserva/pedido, menú/productos en el perfil; Local Services Ads si aplica.

- Tienda online / catálogo
  - Imprescindible: Merchant Center (listados gratis) + Product/Offer; monitoriza Merchant listings en GSC.
  - Recomendable: inventario local (si hay tiendas físicas), Google Ads (Search/PMax).

- Tráfico EEE
  - Imprescindible: Consent Mode v2 (vía CMP/GTM).
  - Recomendable: remarketing avanzado con consentimiento.

## Anexo: Esenciales para negocio local (paso a paso)

1) Reclama y verifica tu Business Profile. Añade categorías primaria/secundaria, horarios (incl. festivos), servicios, fotos y una breve descripción “Desde la empresa”. Sin apartados postales/oficinas virtuales.

2) Si te desplazas a clientes, define áreas de servicio y oculta la dirección en el perfil.

3) Consigue y responde reseñas. La calidad y frescura influyen en la prominencia; responder demuestra atención.

4) Añade acciones útiles: enlaces de reserva/pedido, menú/productos. Nota: chat/llamadas integrados se retiraron en 2024; considera un enlace de WhatsApp/SMS donde esté disponible.

5) (Opcional) Vincula GBP ↔ Google Ads para mostrar dirección e indicaciones en anuncios.

6) Revisa “Rendimiento” en GBP (búsquedas, vistas, llamadas, indicaciones, clics al sitio).

## Anexo: Complementos e‑commerce (también útiles en local)

- Visibilidad gratuita: sincroniza productos con Merchant Center para listados gratis en Search/Shopping/Maps/Images/YouTube/Lens/Gemini.
- Datos estructurados: implementa Product/Offer con precio, disponibilidad, envío y devoluciones; valida con Rich Results Test; monitoriza problemas en GSC Merchant listings.
- Inventario local / listados locales gratis: muestra stock cercano y fomenta visitas en tienda.

## Anexo: Experiencia de página e higiene técnica

- Core Web Vitals: LCP ≤ 2.5s, INP ≤ 200ms, CLS ≤ 0.1. Usa PSI para datos de campo (CrUX) y laboratorio (Lighthouse).
- Sitemap: genera y envía un XML estándar; cuida el enlazado interno.
- Robots e indexación: robots.txt controla el rastreo, no la indexación; usa noindex o autenticación para mantener páginas fuera de Search.
- Soluciona páginas concretas con Inspección de URL (probar en vivo, solicitar indexación).
- Ante volatilidad, revisa Search Status Dashboard (core updates/incidentes).

## Anexo: Investigación y planificación (flujo ligero)

- Empieza con Trends para vocabulario, estacionalidad y geografía; compara hasta 5 grupos de términos.
- Usa Keyword Planner para dimensionar demanda y encontrar temas relacionados (algunas funciones requieren completar la cuenta de Ads).
- Para calidad de contenido, sigue “helpful, reliable, people‑first content” y Search Essentials.

## Checklist de una página (sitio nuevo)

Día 1–2
- Verifica Search Console (archivo HTML, DNS TXT o meta). Envía sitemap.xml. Inspecciona URLs clave.
- Instala GA4 (gtag o vía GTM). Configura 1–3 conversiones (lead, compra, clic de llamada).
- Ejecuta PageSpeed Insights en home y plantillas principales; anota problemas de LCP/INP/CLS.

Semana 1
- Business Profile: reclamar/verificar; categorías, horarios (incl. festivos), áreas de servicio (si aplica), descripción, fotos; añade enlaces de reserva/pedido si aportan. Inicia plan de reseñas.
- Si vendes online: añade Product/Offer; valida en Rich Results Test; abre Merchant listings en GSC.

Mes 1
- Crea un dashboard en Looker Studio mezclando GA4 (engagement/conversiones) + GSC (consultas/posiciones).
- Si atiendes al EEE, despliega Consent Mode v2 con tu CMP vía GTM.
- Considera Local Services Ads (oficios) o una prueba pequeña de Search Ads; vincula Ads ↔ GBP.

## Glosario

- Tag — pequeño fragmento de código que carga el sitio (Analytics, Ads, etc.). GTM ayuda a gestionarlos sin despliegues de código.
- Datos estructurados / schema — información legible por máquinas (JSON‑LD recomendado) que habilita rich results (p. ej., precio/valoraciones). Prueba con Rich Results Test.
- Core Web Vitals (CWV) — LCP, INP, CLS. Aspira a los umbrales “buenos”.
- Sitemap — lista de URLs importantes; envíala en GSC.
- Robots.txt — archivo que gestiona rastreo; no bloquea la indexación (usa noindex o auth para ello).

## Reflexión final

Google sigue siendo la puerta de entrada principal a la web, especialmente para descubrimiento local. Si ganas visibilidad (contenido útil, sitio sano), mides bien (GA4/GSC), mejoras la experiencia (CWV) y enriqueces resultados (datos estructurados, Business Profile, Merchant Center), cubres el 80/20 del crecimiento sostenible; los anuncios y reporting avanzado pueden venir después.
