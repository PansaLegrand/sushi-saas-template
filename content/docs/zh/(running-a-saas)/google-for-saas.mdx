## 为什么 Google 对 SaaS 很重要

Google 是人们提问并发现解决方案的地方。对 SaaS 来说，在 Google 上获胜有两点关键价值：

- 随时间积累的高质量搜索流量。
- 了解用户需求与发现路径，从而改进页面、内容和引导。

不需要“小技巧”。需要的是资格（可被收录）、可发现性、良好的页面体验和有用的内容。Google 自家的工具会告诉你要修什么，以及用户到站后的行为。

## 我们如何使用 Google（工具清单）

### Search Console 与 Google Analytics（GA4）的区别

- Search Console（search.google.com/search-console）
  - 你在 Google 搜索中的表现：查询、展示量、点击、平均排名。
- 暴露抓取/索引问题、核心网页指标（Core Web Vitals）、站点地图、手动处理、结构化数据资格等。
  - 数据来自 Google 搜索本身（无需埋点）。不显示非搜索流量或站内行为。

- Google Analytics 4（analytics.google.com）
  - 用户在站内/应用内的行为：用户、会话、事件、转化、渠道。
  - 事件模型；需要标签（Google tag 或通过 Tag Manager）。

两者互补：Search Console 解释访客如何从 Google 来；GA4 解释他们到站后做了什么。还可以在 Looker Studio 中一起可视化。

### 其它重要的 Google 入口

- Google 商家资料（GBP）：如果服务特定区域，提升地图/“附近”搜索可见度。
- Merchant Center + 产品结构化数据：获取商品富结果/Shopping 的资格（电商相关）。
- Trends + 关键词规划师：了解需求、季节性与查询主题。
- PageSpeed Insights + Search Console 的 Core Web Vitals：真实用户的性能信号。

## 实用增长手册（2025）

1) 先满足基础（资格与政策）
- 遵循 Search Essentials 与垃圾信息政策（禁止操纵性外链、为了排名而批量生成无用内容等）。
- 产出有帮助、可靠、以人为本的内容；Google 的系统会奖励“有用性”，不在乎由谁/如何生成。

2) 确保页面可被发现与收录
- 在 Search Console：修复“页面索引”报告中的问题，提交干净的 XML 站点地图，并确认 robots.txt/noindex 未阻止关键页面。
- 使用“URL 检查”查看具体 URL，并在需要时请求重新抓取。

3) 做好页面体验（Core Web Vitals）
- 目标：LCP ≤ 2.5s、INP ≤ 200ms、CLS ≤ 0.1（以 75 分位计）。
- 使用 PageSpeed Insights，并在 Search Console 中监控 Core Web Vitals。注意：INP 已在 2024 年 3 月取代 FID。

4) 正确添加结构化数据
- 遵循通用指南，并用 Rich Results Test 进行测试。
- 使用 Search Gallery 实现与你相关的类型（Product、Event、Article、Local Business 等）。
- 近期变化：HowTo 与大多数 FAQ 富结果已弃用/收紧；支持产品变体标注。

5) 加强信息架构
- 描述性、干净的 URL；可抓取的链接与清晰锚文本；正确使用 canonical；多语言站点实现 hreflang。

6) 本地可见度（如适用）
- 设置并优化 Google 商家资料（营业时间、分类、照片、服务）。
- 本地排名因素：相关性、距离、知名度；新近的好评有帮助。

7) 电商入口（如售卖在线商品）
- 实施 Product 与 Merchant 列表的结构化数据；在 Search Console 中监控 Merchant listings 报告。
- 注册 Merchant Center 账号，让商品可在 Google（Search/Shopping/Maps/YouTube）展示。可选添加退货政策标注。

8) 研究需求与主题
- 使用 Google Trends 观察关注度和季节性；使用关键词规划师发现并验证查询主题（完整访问需绑定账单的 Google Ads 账号）。

9) 量化与迭代
- 在 Search Console：用 Performance（Search/News/Discover）看查询、CTR、排名；关注 Crawl Stats 和 Search Status Dashboard。
- 在 GA4：定义转化事件（线索、购买），对比拉新与参与。如果在 EEA 运营，实施 Consent Mode v2。
- 构建一个 Looker Studio 报表，融合 Search Console + GA4 跟踪进展。

## 快速对照（备忘）

- 哪些查询带来流量？CTR/排名如何？
  - Search Console → Performance（Search/News/Discover）
- 这个页面被收录了吗？原因是什么？
  - Search Console → URL 检查 / 页面索引
- 页面对用户是否足够快/好用？
  - PageSpeed Insights + Search Console → Core Web Vitals
- 我的商品是否具备富结果/Shopping 资格？
  - Search Console → Merchant listings + Rich Results Test
- 访客到站后做了什么？
  - GA4（事件/转化）

## 常见误区

- 购买/参与操纵性外链，或批量产出无用内容（违反垃圾政策）。近期更新针对“过期域名滥用”“规模化内容滥用”“站点声誉滥用”。
- 依赖已弃用的富结果类型（如 HowTo；大多数 FAQ）。
- 忽视 INP（已在 2024 年 3 月取代 FID）。

## 关键概念：“标签”是什么？

在网站分析/营销语境中，“标签（tag）”通常指加入站点的一小段 JavaScript，用来向某服务（Analytics、Ads 等）发送数据。

- GA4 → 需要标签。示例：带有 Measurement ID（G‑XXXXXXX）的 `<script>`，用于跟踪浏览、事件、转化。
- Google Tag Manager（GTM）→ 容器系统，可在不改动站点代码的情况下管理多种标签（Analytics、Ads、再营销等）。
- Search Console → 不需要标签。只需一次性验证站点所有权（DNS TXT、HTML 文件或在 `<head>` 中添加 `<meta>`），后续搜索/索引数据由 Google 端采集。

## 如何注册 Google 商家资料（本地 SEO）

如果你服务某个地域或想提升地图可见度：

1. 访问 https://www.google.com/business/ 并登录。
2. 填写商家信息：名称、类别、地址或服务区域、联系方式、营业时间、照片。
3. 验证（明信片、电话、邮件或视频）。
4. 优化：保持信息最新，定期添加照片，邀请好评并回复，使用“帖子”发布更新/优惠。

## 按步骤搭建（新站）

1) Search Console
- 验证所有权（推荐 DNS TXT），提交站点地图，查看“页面索引”。
- 修复阻塞（noindex、robots.txt、canonical），按需请求重新抓取。

2) GA4（可选用 GTM）
- 添加 GA4 标签（或经由 GTM 安装）。定义转化事件（注册、购买等）。
- 在 EEA 运营需实施 Consent Mode v2。

3) Google 商家资料（若面向本地）
- 创建/认领资料，完成验证并完善关键字段（营业时间、服务）。开始积累评价。

4) 可选：Merchant Center（如发布商品）
- 添加 Product/Merchant 结构化数据，连接 Merchant Center，并在 Search Console 中监控资格。

5) 报表
- 在 Looker Studio 构建基础仪表板，整合 Search Console + GA4，跟踪查询、CTR、转化与页面表现。

## 附录：必备工具箱（速览）

- Google Search Console（GSC）— SEO 控制室
  - 作用：查询/CTR/排名、收录、Core Web Vitals、富结果资格；提交站点地图；请求索引；Merchant/Product 报告。
  - 前 10 分钟：验证所有权（HTML 文件、DNS TXT 或 meta；GA4/GTM 也可），提交站点地图，检查/索引关键 URL。售卖商品则查看 Merchant listings 与 Product snippets。

- Google 商家资料（GBP）— 搜索/地图的本地信息
  - 作用：发布名称/地址/电话（NAP）、营业时间、类别、照片、服务/商品、预约链接；收集评论；展示本地指标。
  - 提示：内置聊天/通话记录已于 2024‑07‑31 下线；部分地区可加 WhatsApp/SMS 链接替代。资格要求与客户有当面接触（不允许邮政信箱/虚拟办公室）。服务区域型应隐藏地址并设置服务范围。

- Google Analytics 4（GA4）— 事件模型的度量
  - 作用：网站/应用的流量、事件、转化；与 Ads/BigQuery 集成。
  - 前 10 分钟：创建属性，安装 Google 标签或经由 GTM 部署，定义关键转化。在 EEA 需实施 Consent Mode v2（通常通过 CMP + GTM）。

- Google Tag Manager（GTM）— 无需发版管理标签
  - 作用：单一容器部署/更新 Analytics/Ads/同意相关标签。

- PageSpeed Insights（PSI）与 Core Web Vitals — 性能与体验
  - 作用：实测（CrUX）+ 实验室（Lighthouse）数据，优化 LCP/INP/CLS。目标：LCP ≤ 2.5s、INP ≤ 200ms、CLS ≤ 0.1。INP 于 2024 年取代 FID。

- Merchant Center — 免费商品展示面
  - 作用：在 Search/Shopping/Maps/Images/YouTube/Lens/Gemini 免费展示；广告可选。支持商品 Feed 或结构化数据。开发/SEO 提示：添加 Product/Offer（运费/退货、价格/库存），在 GSC 监控 Merchant listings。

- Google Ads — 付费增长（可选）
  - 作用：Search/Performance Max/YouTube/Maps；对某些服务行业可用 Local Services Ads（按线索计费，含审核）。与商家资料互联用于地址扩展。

- Looker Studio — 报表看板
  - 作用：免费仪表板；连接 GA4 + GSC 联合可视化。

- 研究工具 — Trends 与 关键词规划师
  - 作用：发现上升话题并比较术语（Trends）；获取关键词思路与量级（Planner；部分功能需完整开通 Ads 账号）。

## 附录：按场景“必备 vs 可选”

- 任何网站上线
  - 必备：Search Console（验证、站点地图、URL 检查）、GA4（基础转化）、PSI/CWV 修复。
  - 可选：Looker Studio 看板；合并查看 GSC ↔ GA4。

- 本地门店/服务区域型
  - 必备：商家资料（NAP、营业时间、类别、照片、评论）；本地排序因素（相关性、距离、知名度）。
  - 可选：预约/下单链接、菜单/商品；合格行业的 Local Services Ads。

- 在线商店 / 目录
  - 必备：Merchant Center（免费展示）+ Product/Offer；在 GSC 监控 Merchant listings。
  - 可选：本地库存（如有门店）、Google Ads（Search/PMax）。

- EEA 流量
  - 必备：Consent Mode v2（通过 CMP/GTM）。
  - 可选：基于同意的高级再营销。

## 附录：本地商家要点（分步）

1) 认领并验证商家资料。添加主/副类别、营业时间（含节假日）、服务、照片，“商家介绍”短描述。不得使用邮政信箱/虚拟办公室。

2) 上门服务型请设置服务范围并在资料中隐藏地址。

3) 收集并回复评论。高质量、近期评论影响知名度；及时回复体现活跃。

4) 添加有用操作：预约/下单链接、菜单/商品。提示：内置聊天/通话记录已退场；可在支持地区添加 WhatsApp/SMS 链接。

5) （可选）打通商家资料 ↔ Google Ads，在广告中显示地址和路线。

6) 在商家资料“效果”中查看搜索、浏览、来电、路线、网站点击。

## 附录：页面体验与技术卫生

- Core Web Vitals 目标：LCP ≤ 2.5s、INP ≤ 200ms、CLS ≤ 0.1。用 PSI 同看实测（CrUX）与实验室（Lighthouse）。
- 站点地图：生成并提交标准 XML；保持内部链接清晰。
- Robots 与收录：robots.txt 控制抓取而非收录；保持出索引请使用 noindex 或鉴权。
- 单页排障：用 URL 检查（实时测试、请求收录）。
- 排名波动时，查看 Search Status Dashboard（核心更新/事故）。

## 新站一页式清单

第 1–2 天
- 验证 Search Console（HTML 文件、DNS TXT 或 meta），提交 sitemap.xml，检查若干关键 URL。
- 安装 GA4（gtag 或经由 GTM）。配置 1–3 个转化（线索、购买、拨打点击）。
- 用 PageSpeed Insights 测试首页/主要模板；记录 LCP/INP/CLS 关键问题。

第 1 周
- 商家资料：认领/验证；设置类别、营业时间（含节假日）、服务范围（如适用）、描述、照片；按需添加预约/下单链接。启动评价计划。
- 若售卖商品：实现 Product/Offer；用 Rich Results Test 验证；在 GSC 打开 Merchant listings 报告。

第 1 个月
- 搭建小型 Looker Studio 看板，融合 GA4（参与/转化）+ GSC（查询/排名）。
- 面向 EEA 时，通过 CMP/GTM 部署 Consent Mode v2。
- 视情况考虑 Local Services Ads（服务行业）或小规模 Search Ads；联通 Ads ↔ 商家资料。

## 术语表

- 标签（Tag）— 站点加载的一小段代码（Analytics、Ads 等）。GTM 可在不发版的情况下管理标签。
- 结构化数据 / schema — 机器可读信息（推荐 JSON‑LD），可解锁富结果（如价格/评分）。用 Rich Results Test 测试。
- Core Web Vitals（CWV）— LCP、INP、CLS。以“良好”阈值为目标。
- 站点地图 — 重要 URL 列表；在 GSC 提交。
- Robots.txt — 管理抓取的文件；不用于阻止收录（请使用 noindex 或鉴权）。

## 结语

Google 仍是大多数用户通往网页的首要入口，尤其在本地发现上。只要赢得可见度（有用内容、健康站点）、正确度量（GA4/GSC）、修复体验（CWV）、并丰富结果（结构化数据、商家资料、Merchant Center），就能覆盖可持续增长的 80/20；广告与高级报表可随规模增长逐步加入。
