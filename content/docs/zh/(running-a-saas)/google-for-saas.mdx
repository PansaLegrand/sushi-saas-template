## 为什么 Google 对 SaaS 很重要

Google 是人们提问并发现解决方案的地方。对 SaaS 而言，在 Google 上获得可见性有两大价值：

- 随时间复利增长的高质量流量；
- 洞察用户需求与发现路径，从而改进内容、页面与引导。

截至 2025 年中，Google 约占全球网络搜索的 90%（移动 + 桌面）。要进入自然结果本身是免费的，你需要让网站满足 Search Essentials：以人为本的内容、可抓取并对移动友好、良好的页面体验，且避免操纵性做法。

- 搜索引擎份额：https://gs.statcounter.com/search-engine-market-share
- Search Essentials（策略与技术要点）：https://developers.google.com/search/docs/essentials

不需要“小技巧”，需要的是：可收录、可发现、好体验、有用内容。Google 自家工具会告诉你该修什么、用户落站后如何行为。

## Google 工具箱（助力 SaaS 增长）

### Google Search Console（GSC）

你的 SEO 中控室：展示查询、曝光、点击和排名；提示抓取/索引问题；报告 Core Web Vitals；校验结构化数据资格。数据来自 Google 搜索（无需埋点）。可提交站点地图、检查 URL，并在必要时请求重新抓取。

- GSC：https://search.google.com/search-console

### Google Analytics 4（GA4）

站内行为分析：跨渠道的用户、会话、事件、转化。需要标签或 Google Tag Manager，采用事件模型。与 GSC 互补，解释访客到站后做了什么。

- GA4：https://analytics.google.com

### Google 商家资料（GBP）

若在本地有可见性诉求（办公室、培训、区域聚焦），GBP 帮你出现在搜索/地图中。排序基于相关性、距离与知名度。保持类目、营业时间等信息准确，添加照片并管理评价。

- 提升本地排名：https://support.google.com/business/answer/7091

### Merchant Center + 产品结构化数据

若你销售产品（硬件、周边等），Merchant Center 可在多种 Google 场景里获取免费商品展示。即便不用 MC，Product schema 也可解锁商品富结果。2024 年 Google 扩展了产品变体的标注支持。

- 免费商品展示概览：https://support.google.com/merchants/answer/13889434
- 变体支持（schema）：https://www.schemaapp.com/schema-markup/google-expands-structured-data-support-for-product-variants/

### Trends 与关键词规划师

用 Trends 识别上升主题与季节性；用关键词规划师获取思路与大致量级（通常需完成 Google Ads 账户与计费配置，但不强制投放）。

- 关键词规划师：https://support.google.com/google-ads/answer/7337243

### PageSpeed Insights（PSI）与 Core Web Vitals（CWV）

CWV 关注真实用户的速度与体验：LCP（加载）、INP（交互；2024 年起替代 FID）与 CLS（稳定）。目标：LCP ≤ 2.5s、INP ≤ 200ms、CLS &lt; 0.1。用 PSI 做诊断，用 GSC 监控整体表现。

- CWV 说明：https://www.tuurbo.ai/speed-performance/what-are-core-web-vitals/

### Looker Studio

构建可视化报表，融合 GSC + GA4（及更多）以关联 SEO 点击与注册/转化，并跟踪进展。

- Looker Studio：https://lookerstudio.google.com

## 2025 实战手册（排名与增长）

1）打好基础（资格与政策）
- 遵循 Search Essentials；避免垃圾策略（链接方案、低质量规模化内容）。2024 年 3 月 Google 打击了过期域滥用、规模化内容滥用与站点声誉滥用。
- 生产对人有用、可靠的内容。长期来看，专业与实用会获胜。

2）保证可发现与可索引
- 索引：用 GSC 报告修复问题，确保关键页面被索引。
- 站点地图：提交 XML sitemap 并保持更新。
- URL 检查：逐页排查，并为重要新/大改页请求索引。
- 避免误挡：robots.txt 控制抓取而非索引；敏感内容请用 noindex 或鉴权。

3）优化页面体验（Core Web Vitals）
- 目标：LCP ≤ 2.5s、INP ≤ 200ms、CLS &lt; 0.1（真实用户 p75）。
- 用 PSI 诊断（大图、长任务、阻塞脚本）。优先处理影响模板的系统性问题。
- 提示：INP 已替代 FID；确保首个交互之后也持续流畅。

4）实现结构化数据（schema）
- 遵循指南并用 Rich Results Test 校验。
- 重点类型：Article（博客/文档）、Event（活动/网络研讨会）、Product 或 SoftwareApplication（套餐/功能）、Organization（全站信息）。
- 变化：HowTo 已弃用；FAQ 仅在少数权威站点显示——不要依赖其获取流量。
- 产品变体（如适用）可用 ProductGroup‑style 标注。

5）强化站点结构与内链
- URL 清晰；可抓取链接与语义锚文本；正确的 canonical；多语言用 hreflang。
- 避免孤立页面；修复死链；层级应符合导航逻辑。

6）本地 SEO（如适用）
- 创建/认领 GBP，选择准确类目，补全营业时间（含节假日）、照片与简介。
- 鼓励并回复评价；信息常新。内置聊天/通话记录已于 2024 年下线；部分地区可链接 WhatsApp/SMS。

7）电商相关（如有产品销售）
- 实施 Product schema（价格、库存、运费/退货等），并用 Rich Results Test 校验。
- 配置 Merchant Center 并开通免费商品展示；在 GSC 监控 Merchant listings 报告。
- 若有门店，可探索本地免费商品/本地库存。

8）持续研究需求与主题
- 用 Trends 看词汇与季节性；关键词规划师/第三方工具看量级与相关词。
- 同时覆盖强意图（定价/对比）与教育型主题，映射产品价值。

9）度量并迭代
- 在 GSC：看 Performance（查询/CTR/排名）、索引/覆盖与抓取统计；关注 Search Status Dashboard 的更新与事件。
- 在 GA4：定义并跟踪转化（注册、演示、购买）。审视参与度与路径。服务 EEA 时实现 Consent Mode v2。
- 在 Looker Studio：融合 GSC + GA4 关联 SEO 点击与注册，并定位改进点（如点击高但转化低）。

- 质量/垃圾更新：https://blog.google/products/search/google-search-update-march-2024/
- 搜索状态面板：https://status.search.google.com/summary

## 速查（Cheat Sheet）

- 哪些查询带来流量？CTR 与排名？
  - Search Console → Performance（Search/News/Discover）
- 页面 X 是否被索引？若否，原因？
  - Search Console → URL Inspection + Page indexing 报告
- 页面速度/可用性如何？
  - PageSpeed Insights 诊断 + GSC → Core Web Vitals
- 商品页是否具备富结果/Shopping 资格？
  - GSC → Merchant listings/Products；Rich Results Test；Merchant Center feed 状态
- 访客落站后做了什么？
  - GA4（获客、事件、转化；细分 Google organic）

## 常见误区

- 追求“捷径”（链接方案、PBN、薄/自动化内容）。近期反垃圾更新会打压。
- 依赖已弃用/受限的富结果（HowTo/FAQ）。不要围绕它们构建策略。
- 忽视 INP（现行交互性指标）。良好的 FID 可能掩盖糟糕的 INP。
- 移动体验薄弱或移动/桌面内容不一致（mobile‑first）。
- 忽略隐私/同意（EEA）。实现 Consent Mode v2 以在合规前提下尽可能保留测量。
- 把 SEO 当“一次性工程”。基于 GSC/GA4 洞察按季度迭代。

## 关键概念：这里的“标签（tag）”是什么？

在分析/营销中，“标签”是网站加载的 JS 代码片段，用于向某服务（Analytics、Ads 等）发送数据。

- GA4 → 需要标签。示例：带 Measurement ID（G‑XXXXXXX） 的 `<script>`，用于页面、事件与转化跟踪。
- Google Tag Manager（GTM） → 一个容器，可在不改代码的情况下管理多种标签。
- Search Console → 不需要标签。通过 DNS TXT、HTML 文件或 `<meta>` 验证所有权后，Google 在其端收集搜索/索引数据。

## 步骤顺序（新站）

1）Search Console
- 验证所有权（推荐 DNS TXT），提交 sitemap，并检查页面索引。
- 修复阻断（noindex、robots.txt、canonical），必要时请求重新抓取。
- 检查重点 URL，并在重大更新后请求索引。

2）GA4（+ 可选 GTM）
- 安装 GA4 标签（或经 GTM）。定义 1–3 个核心转化（注册、购买、演示）。
- 服务 EEA 时实现 Consent Mode v2（通常经 CMP + GTM）。

3）Google 商家资料（若本地化）
- 创建/认领并验证，完善（类目、时间、简介、照片）。启动评价计划。

4）Merchant Center（若有商品）
- 实施 Product/Merchant；配置 MC；开启免费商品展示；在 GSC 监控 Merchant listings。

5）报表
- 用 Looker Studio 融合 GSC + GA4 监控查询、CTR 与转化。

## 接下来做什么

- 决定关键场景（仅 Search？本地？商品？）。
- 若完全在线且面向全球，优先 SEO + GA4 度量；GBP 可选。
- 若涉及区域/线下要素，优化 GBP 与本地 SEO 基础。
- 若销售附加品/硬件/周边，结合 Merchant Center + schema 提升可见度。
- 所有人都应关注 Google 更新；夯实基础能更易适应变化。

## 术语表

- Google Search Console（GSC）— 搜索分析与索引健康；无需站内标签。
- Google Analytics 4（GA4）— 事件驱动分析；需标签或 GTM；跟踪参与与转化。
- Google 商家资料（GBP）— 你的搜索/地图名片；优化本地可见度与信任。
- Google Merchant Center（GMC）— 商品 feed 用于免费展示与 Shopping；监控资格与政策。
- 结构化数据 / schema — JSON‑LD 标注，启用富结果；应反映页面真实内容。
- Core Web Vitals（CWV）— LCP、INP、CLS 的良好阈值。
- XML sitemap — URL 清单，助力发现；在 GSC 提交。
- robots.txt — 控制抓取（非索引）。私密页用 noindex/鉴权。
- Google Tag Manager（GTM）— 单一容器管理多标签，无需频繁发版。
- Consent Mode — 让 Google 标签依据同意状态自适应；在 EEA 至关重要。
- Looker Studio — 免费看板，融合 GSC/GA4 展示 KPI。
